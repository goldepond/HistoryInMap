# 작업 지시서 / 초기 컨셉 — 연도별 세계사 지도 웹사이트

> 본 문서는 프로젝트의 **초기 컨셉과 작업 범위**를 고정하기 위한 기준 문서다.
> 구현 세부는 [README.md](../README.md) 를, 데이터 출처/라이선스는 같은 README의 *데이터 출처* 절을 참고한다.

---

## 1. 프로젝트 목표

연도(시대)를 기준으로 세계사를 정리하는 **지도 중심 반응형 웹사이트**를 만든다.
사용자가 타임라인을 움직이면, 같은 시대에 유럽·남미·동아시아 등 여러 지역에서
일어난 사건을 지도 위에서 **병렬적으로 한눈에 비교**할 수 있어야 한다.

## 2. 핵심 제약 (반드시 지킬 것)

| 제약 | 내용 |
| --- | --- |
| **완전 무료** | 유료 API·결제·사용량 한도가 있는 서비스 금지 |
| **자체 호스팅** | 타일 서버·API 키 없이 정적 호스팅(GitHub Pages / Cloudflare Pages / Netlify)에 올라가는 **정적 사이트** |
| **반응형** | 데스크톱·모바일 모두 자연스럽게 동작 |
| **외부 의존 최소화** | 지도 데이터·라이브러리를 프로젝트 내부에 포함하거나 같은 도메인에서 서빙 |

> 이 프로젝트는 위 제약을 모두 만족한다: Leaflet·경계 데이터·사건 데이터가 모두 리포지토리에 **로컬 번들**되어 있고, 런타임에 외부 CDN/타일 서버를 호출하지 않는다.

## 3. 지도 데이터 소스

- **저장소**: <https://github.com/aourednik/historical-basemaps> (GPL-3.0)
- **형식**: 시대별 GeoJSON (`world_bc2000.geojson`, `world_1492.geojson` … 기원전부터 현재까지 수십 시점)
- **연도 목록**: 리포지토리 루트의 `index.json` 에 사용 가능한 연도와 각 파일의 국가 목록이 정리됨 → 타임라인 눈금을 동적으로 채우는 데 사용
- **폴리곤 속성 필드** (스타일링 활용):
  - `NAME` — 국가/지역명 (라벨)
  - `SUBJECTO` — 종주국 (지역 색칠/그룹핑)
  - `PARTOF` — 상위 문화권
  - `BORDERPRECISION` — 데이터 제작자의 **경계 신뢰도**(1=대략적 추정, 3=확실히 기록된 경계, 2는 거의 안 씀). *국제법/표준 정의가 아니라 사료에 대한 제작자 판단*이라 다소 주관적. 모호한 경계는 흐림/점선으로 표현
  - `ABBREVN` — 약칭 (부가)
- **도시 데이터**: `places.geojson` (`inhabitedSince` / `inhabitedUntil` 연도 필드로 시대 필터링 가능) — *향후 레이어*
- **라이선스**: GPL-3.0. 웹 게시는 자유. **데이터를 수정·재배포하면 동일 라이선스로 공개**해야 함 (README에 명시)

## 4. 기술 스택

- **렌더링**: **Leaflet** 채택 (구현 난이도·줌/팬 부드러움 우선). D3는 커스텀 투영이 필요해질 때 재검토.
- **빌드**: 바닐라 JS (ES Modules). 번들러·프레임워크 없음.
- **라이브러리 제공**: Leaflet 1.9.4 를 `vendor/leaflet/` 에 **로컬 번들** (외부 CDN 미사용).

## 5. 기능 명세

| # | 기능 | 설명 | MVP |
| --- | --- | --- | --- |
| 1 | **타임라인/연도 슬라이더** | `index.json` 연도로 눈금 구성. 슬라이더 이동 시 해당 시대 `world_*.geojson` 로드 후 경계 표시 | ✅ |
| 2 | **지역 병렬 비교** | 같은 연도 화면에서 유럽·남미·동아시아 등 사건을 동시에 노출 (지역 패널 + 지도 마커 연동) | ✅ |
| 3 | **사건 레이어** | 경계 위에 사용자 세계사 사건 데이터를 별도 레이어로 얹어 연도별 필터링·마커/팝업 표시 | ✅ |
| 4 | **마커 인터랙션** | 마커 클릭 시 사건 상세(제목·연도·지역·설명) 팝업 | ✅ |
| 5 | **경계 스타일링** | `SUBJECTO` 로 지역 색 구분, `BORDERPRECISION` 으로 모호한 경계 시각 처리 | ✅ |

## 6. 사용자 사건 데이터 모델

별도 JSON 파일(`data/events.json`)로 관리하며 콘텐츠 추가가 쉬워야 한다.

```json
{
  "year": 1492,
  "endYear": null,
  "region": "남미",
  "title": "콜럼버스 신대륙 도착",
  "lat": 23.1,
  "lng": -82.3,
  "description": "..."
}
```

- **단일 연도 사건**(`endYear: null`)과 **기간 사건**(`year`~`endYear`) 모두 표현 가능
- `region` 필드로 지역 패널 분류 (유럽 / 남미 / 동아시아 / …)

## 9. 주의사항

- 데이터 좌표계는 **WGS84(EPSG:4326)**. 세계 단위 표시 시 면적 왜곡이 적은 투영(Mollweide 등)은 향후 과제.
- 일부 시대는 국경이 학술적으로 불확실 → **"참고용, 학술 인용 전 교차검증 필요"** 안내 문구를 사이트에 상시 노출.
- GeoJSON 파일이 크므로 연도 전환 시 **필요한 파일만 lazy-load**(+ 메모리 캐시).

## 10. 진행 순서

1. **MVP** — 연도 슬라이더 + 경계 로딩 + 샘플 사건 마커 (← 현재 완료 지점)
2. 지역 병렬 패널 + `SUBJECTO`/`BORDERPRECISION` 스타일링 (← 현재 완료 지점)
3. 향후 — `places.geojson` 도시 레이어, 기간 사건 타임바, 투영 개선, 전체 연도 데이터 벤더링

---

### 현재 구현 상태 (이 리포지토리)

MVP + 2단계(지역 병렬 패널 + 경계 스타일링)까지 구현 완료.
경계 데이터는 14개 시점(기원전 2000년 ~ 2010년)을 벤더링했고,
`scripts/fetch-borders.mjs` 로 원본 리포지토리의 전체 시점을 내려받아 확장할 수 있다.

## 11. v2 확장 — 미시적 사건 + 편집 (구현 완료)

초기 컨셉의 "국경 뷰어"를 넘어, **사건의 미세 위치·이동 경로를 직접 저작**하는 도구로 확장했다.

### 11.1 추가 요구사항 (사용자 확정)

| # | 요구 | 결정 |
| --- | --- | --- |
| A | 타임라인 간격이 너무 넓음 | **2단계 타임라인** — 평소 전 세계 10년 단위, *장면(Scene)* 진입 시 해당 지역만 포커스해 **연 단위** |
| B | 사건의 미시적 움직임(콜럼버스 항해, 카이사르 원정 경로) | 사건에 **경로(route)** + 순서 있는 경유지 + **이동 애니메이션** 추가 |
| C | 국가 구분보다 정확한 지리 | **Natural Earth** 물리 지형(해안선·강·호수)을 배경에 추가 (퍼블릭 도메인, 오프라인) |
| D | 콘텐츠를 자유롭게 추가 | 지도 위 그리기(Geoman) + JSON 직접 편집 **둘 다**, GeoJSON/JSON 내보내기 |
| E | 국경 오류 수정 | 국경 폴리곤을 편집 모드에서 꼭짓점 드래그로 수정 → `world_*.geojson` 내보내기 |

### 11.2 데이터 모델 v2

- **사건**: `geometry` = `point` | `route` | `area`. 좌표는 GeoJSON 순서 `[lng, lat]`.
  - `route` 는 `waypoints`(이름·날짜 포함) 또는 `coordinates`, `animate: true` 로 재생.
  - `start`/`end` 로 단일·기간 사건. 레거시 `year`/`endYear`/`lat`/`lng` 자동 변환.
- **장면(`scenes.json`)**: `bounds`(지역 포커스), `start`/`end`/`step`(연 단위), `borderYear`(표시할 국경 스냅샷).

### 11.3 편집·내보내기

- 정적 사이트 제약을 지키기 위해 **서버 저장 없이** 브라우저에서 GeoJSON/JSON 을 **다운로드** → 사용자가 리포지토리에 커밋.
- 라이브러리: Leaflet-Geoman (MIT, 로컬 번들).

### 11.4 제약 유지 확인

완전 무료 / 자체 호스팅 / 외부 호출 0 / 반응형 — v2에서도 모두 유지.
Natural Earth(퍼블릭 도메인)·Geoman(MIT)을 로컬 번들해 외부 의존을 늘리지 않았다.

### 11.5 향후 과제

- 경유지 날짜 기반으로 슬라이더 위치 ↔ 경로 진행률 연동(현재는 ▶ 버튼 재생)
- `places.geojson` 도시 레이어, Mollweide 투영, 장면별 사건 자동 묶기 UI

## 12. v2.1 — 실시간 저장 서버 (구현 완료)

"다운로드 후 파일 교체"가 실사용에 불편하다는 피드백에 따라, **편집 즉시 저장**되는 백엔드를 추가했다.

- **`server.mjs`** — Node 내장 모듈만 사용(외부 의존성 0). 정적 파일 서빙 + 저장 API:
  - `PUT /api/events` → `data/events.json`
  - `PUT /api/scenes` → `data/scenes.json`
  - `PUT /api/borders/<world_*.geojson>` → 해당 국경 파일 (파일명 화이트리스트로 경로 탈출 차단)
- 클라이언트는 사건 **추가·수정·이동·삭제**, 국경 수정 시 자동으로 서버에 저장. 서버가 없으면 **다운로드로 폴백**.
- `0.0.0.0` 바인딩 → 같은 네트워크의 누구나 편집 가능(개인용, **인증 없음**).

### 제약 변경 (의도된 트레이드오프)

원 컨셉의 "순수 정적 사이트" 제약은 **편집·저장을 쓸 때**는 완화된다(작은 자체 서버 필요).
다만 **읽기 전용 공개본은 여전히 정적 호스팅** 그대로 가능하며, 그때 편집은 ‘내보내기’로만 동작한다.
서버도 Node 내장 모듈만 쓰므로 추가 설치·비용·외부 의존은 없다(완전 무료 유지).

## 13. v3 — 콘텐츠를 마크다운 파일로 (위키/Terraform식, 구현 완료)

"모든 것을 편집(시대·내용 모두)" + "대규모 문서는 폼 클릭보다 마크다운 작성" + "위키/Terraform식 파일 관리"
피드백을 반영해, 데이터의 **원천을 JSON에서 `content/*.md` 파일로 이관**했다.

### 13.1 원칙

- **엔티티 하나 = 파일 하나** (`content/events|scenes|eras/<id>.md`). 상단 프론트매터(메타) + 마크다운 본문.
- **진실의 원천은 파일.** 사람이 직접 써도 되고(Terraform식, git 버전관리), UI에서 원문(.md)을 그대로 편집해도 된다(위키식).
- **모든 종류가 편집 대상**: 사건뿐 아니라 **시대(타임라인 눈금)와 장면**도 추가·수정·삭제.

### 13.2 구성요소

- `lib/content.mjs` — 프론트매터 파서/직렬화 + 매니페스트 빌더(서버·스크립트 공용, 의존성 0).
  - 메타 값 규칙: **JSON 파싱 시도 → 실패 시 문자열**. 좌표 배열은 인라인 JSON, 한글은 따옴표 없이.
- `server.mjs` — `GET /api/content`(매니페스트), `PUT/DELETE /api/content/:type/:id`(원문 저장/삭제),
  `PUT …/:id/geometry`(지도 드래그 결과 반영), `GET/PUT /api/borders`(국경).
- 클라이언트 — 매니페스트로 렌더(본문은 `marked`로 MD→HTML), **원문 .md 편집기**(실시간 미리보기),
  **관리 패널**(시대·장면 추가/수정/삭제), 지도 드로잉으로 좌표 자동 채움.
- `scripts/migrate-to-content.mjs` — 구 `data/*.json` → `content/*.md` 1회 이관.

### 13.3 타임라인이 데이터가 됨

시대(eras)가 `content/eras/*.md` 로 관리되므로, **시대 자체를 추가/수정/삭제**하면 타임라인 눈금이 바뀐다.
era = `year` + `borderFile`(표시할 국경 GeoJSON) + `label`.

### 13.4 제약

완전 무료·자체 호스팅·외부 호출 0·반응형 유지. marked(MIT)를 로컬 번들.
읽기 전용 공개는 `content/manifest.json` 으로 정적 호스팅 가능. 본문 HTML 살균은 공개 환경용 향후 과제.

## 14. v4 — 순수 정적 + Git 관리 + 속도 최적화 (구현 완료)

콘텐츠가 `.md` 파일 기반이 된 이상 인앱 편집(서버 의존)은 불필요 → **편집 기능을 전부 제거**하고
**작성은 git으로, 배포는 GitHub Actions로** 자동화했다. 원 컨셉의 "순수 정적 사이트" 제약으로 복귀한다.

### 14.1 변경

- **인앱 편집 제거** — Geoman(드로잉), 문서 편집기, 관리 패널, `/api/*` 저장 레이어, 저장 상태 UI 삭제.
  `js/app.js` 는 읽기 전용 렌더만 남기고, `loadManifest()`는 `content/manifest.json`만 읽는다.
- **속도 최적화**
  - Geoman(약 308KB) 로드 제거.
  - **빌드 시 경계 경량화** — `data/borders-src/`(원본) → `data/borders/`(표시용).
    Douglas–Peucker 단순화 + 좌표 3자리 반올림 + compact JSON ⇒ 약 70% 용량 감소(`scripts/build-borders.mjs`).
  - 기존 캔버스 렌더링 + 레이어 캐시 + 스크럽 디바운스 유지.
- **Git 워크플로** — `.md` 수정 → push → `.github/workflows/deploy.yml` 가
  `build-borders` + `build-manifest` 실행 후 GitHub Pages로 배포.
  매니페스트·경량 경계는 **빌드 산출물(.gitignore)** 이라 커밋하지 않는다.

### 14.2 빌드 파이프라인

```
content/*.md ─(build-manifest)→ content/manifest.json ─┐
data/borders-src/*.geojson ─(build-borders)→ data/borders/*.geojson ─┴→ GitHub Pages
```

로컬 미리보기는 `npm run build` 후 `server.mjs`(정적 전용) 또는 `npx serve`.

## 15. v5 — 정사각 투영 + 지형 토글 + 좌표 피커 (구현 완료)

- **좌표 피커(📍 좌표)** — 기본 OFF. 켜면 커서 좌표 실시간 표시 + 지도 클릭 시 `[lng,lat]` 클립보드 복사 + 그 지점에 핀.
  창작자가 `content` 좌표를 쉽게 얻는 도구. 모드 중엔 국경·사건 pane을 `pointer-events:none`으로 막아 어디서든 좌표가 잡힘.
- **정사각(EPSG:4326) 투영으로 통일** — Leaflet 기본값이던 메르카토르(웹 타일용)는 우리(타일 미사용)에겐 이점이 없고 고위도 면적 왜곡만 큼.
  정사각으로 바꿔 왜곡↓ + 지형 이미지와 픽셀 정렬. 데이터·로직은 lat/lng 기반이라 기능 100% 유지, 지도 옵션(줌/뷰/경계)만 재조정.
- **지형 토글(🏔 지형)** — 기본 OFF. 켜면 **큰 줄기 벡터 지형도**(Natural Earth 지형 구역 268개: 산맥·사막·고원·분지·평원…,
  `data/basemap/terrain-regions.geojson`, 퍼블릭 도메인)를 유형별 색+이름으로 `terrainPane`에 올리고 국경을 살짝 흐리게 → 지형 위에서 국경·경로를 본다.
  벡터라 어느 줌에서도 선명·가볍고 오프라인 유지. *(단일 위성 이미지·온라인 타일안은 "줌인 시 흐림 / 외부 의존" 때문에 폐기.)*
  재생성: `node scripts/build-terrain.mjs`.
- 모두 **순수 클라이언트·정적** 유지. 외부 런타임 호출 0.
